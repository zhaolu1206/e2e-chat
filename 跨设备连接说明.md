# 跨设备连接说明

## 当前限制

由于是纯静态部署，没有信令服务器，跨设备连接有以下限制：

1. **BroadcastChannel** - 只能在同一浏览器的不同标签页之间工作
2. **localStorage** - 只能在同一域名下工作，无法跨设备

## 解决方案

### 方案 1: 使用公共信令服务器（推荐）

部署一个简单的 WebSocket 信令服务器到 Cloudflare Workers，实现真正的跨设备连接。

### 方案 2: 手动输入 Peer ID（临时方案）

两个设备加入房间后，手动输入对方的 Peer ID 来建立连接。

### 方案 3: 使用 URL 参数传递 Peer ID

一个设备生成连接链接，另一个设备打开链接自动连接。

## 临时解决方案

在实现信令服务器之前，可以使用以下方法：

1. **同浏览器多标签页** - 可以正常使用
2. **同一网络下的不同设备** - 需要配置 TURN 服务器
3. **不同网络** - 需要信令服务器

## 建议

要实现真正的跨设备连接，建议：
1. 部署 WebSocket 信令服务器到 Cloudflare Workers
2. 或使用第三方信令服务（如 Firebase、Socket.io 等）
