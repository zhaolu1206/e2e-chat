# 测试指南

## 本地测试步骤

### 1. 启动本地服务器

```bash
cd /home/luke/e2e-group-chat
python3 -m http.server 8000
```

### 2. 测试同浏览器多标签页聊天

1. 打开浏览器，访问 `http://localhost:8000`
2. 在第一个标签页：
   - 输入昵称：`用户1`
   - 输入房间 ID：`test123`（或点击生成随机 ID）
   - 点击"加入聊天室"
3. 打开第二个标签页，访问 `http://localhost:8000`
4. 在第二个标签页：
   - 输入昵称：`用户2`
   - 输入相同的房间 ID：`test123`
   - 点击"加入聊天室"
5. 在两个标签页中发送消息，应该能看到对方的消息

### 3. 测试端对端加密

1. 打开浏览器开发者工具（F12）
2. 在 Network 标签中，查看 WebRTC 连接
3. 在 Console 中，可以看到加密/解密的日志
4. 所有通过 DataChannel 传输的消息都是加密的

### 4. 测试多用户群聊

1. 打开 3-4 个标签页
2. 所有标签页使用相同的房间 ID
3. 每个标签页使用不同的昵称
4. 在一个标签页发送消息，其他所有标签页都应该收到

## 已知限制

1. **同浏览器限制**: 使用 BroadcastChannel 时，只能在同一浏览器的不同标签页之间连接
2. **跨设备需要信令服务器**: 要实现跨设备、跨网络的连接，需要部署 WebSocket 信令服务器
3. **NAT 穿透**: 某些网络环境（如企业防火墙）可能无法建立 P2P 连接

## 故障排除

### 问题：无法连接其他用户

- 检查是否在同一浏览器中打开多个标签页
- 检查房间 ID 是否完全相同
- 检查浏览器控制台是否有错误信息
- 确认网络允许 WebRTC 连接

### 问题：消息无法发送

- 检查连接状态是否为"已连接"
- 检查是否有其他用户在线（在线人数 > 0）
- 查看浏览器控制台的错误信息

### 问题：消息无法接收

- 确认对方已成功加入房间
- 检查 WebRTC 连接状态
- 查看浏览器控制台的错误信息
